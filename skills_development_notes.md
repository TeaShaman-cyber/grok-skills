**Важное замечание по GitHub Connector (17.06.2026):**
Известны проблемы с надёжностью write-операций (commit/push), токенами и отваливанием коннектора. Поэтому использование GitHub/MCP **не может** быть обязательным условием работы со скилами и артефактами.

### Принципы разработки skills (договорённости)

- **Практичность > чистый код** — главное удобство и польза, а не идеальная архитектура.
- **Блоки диагностики обязательны** — особенно для работы с MCP и коннекторами.
- **Прозрачность работы с MCP** — всегда упоминать `grok-mcp check`, `grok-mcp ls`.
- **Сохранение сути** без потери критической структуры.
- **Связь с ТИД** — где уместно (C, Δf, γ, У-вэй, когнитивная навигация).
- **Многоуровневая система якорей**:
  - **Primary (сейчас)**: Репозиторий `grok-skills` на GitHub
  - **Fallback**: Встроенные (bundled) skills от xAI (`mcp`, `skill-creator`, `memory-edit` и др.)
